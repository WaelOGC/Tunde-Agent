# Project map

Scan of the repository tree (excluding `.git`, `node_modules`, `__pycache__`, `.pytest_cache`). Each folder summarizes purpose; every file has a one-line role.

### Project root (`/`)
Repository root: tooling config, compose, packaging, and local dev artifacts.

- `.cursorrules` — Cursor editor rules for AI-assisted development in this repo.
- `.dockerignore` — Exclude patterns for Docker build context.
- `.env` — Local environment variables (secrets); typically gitignored.
- `.env.example` — Template env vars documenting keys for local and Docker runs.
- `.gitattributes` — Git line-ending and binary-file attributes.
- `.gitignore` — Git ignore patterns for Python, Node, env files, caches, and DBs.
- `alembic.ini` — Alembic configuration pointing at migration scripts path.
- `docker-compose.yml` — Compose stack for API, Postgres, migrations, and related services.
- `pyproject.toml` — Python packaging, dependencies, setuptools discovery, pytest markers.
- `README.md` — Project overview, quick start, Docker, and documentation pointers.
- `requirements.txt` — Supplementary Python dependencies for non-editable installs.
- `setup_keys.py` — Helper script for generating or validating API or crypto keys.
- `tunde_dev.db` — SQLite development database file for local backend or agent testing.
